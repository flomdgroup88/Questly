# Оффлайн-режим на iOS (PWA)

## Как это работает

После первого открытия приложения Service Worker скачивает
и кэширует ВСЕ файлы приложения на устройство пользователя.
Railway (или любой другой хостинг) после этого не нужен для работы UI.

```
Первый визит:    Safari → Railway → скачивает JS/HTML/CSS/иконки → кэш на устройстве
Следующие визиты: Safari → Service Worker (на устройстве) → из кэша, без сети
```

Данные (задачи, XP, события) хранятся в localStorage — тоже на устройстве.
Firebase-синк недоступен оффлайн — данные сохранятся и синкнутся при появлении сети.

## Что нужно сделать пользователю (один раз)

1. Открыть приложение в Safari
2. Нажать кнопку "Поделиться" (квадрат со стрелкой вверх)
3. Выбрать "На экран «Домой»"
4. Открыть приложение с экрана домой

После этого работает как нативное приложение, в том числе оффлайн.

## Ограничения iOS

- Safari удаляет кэш PWA если не открывать приложение **более 7 дней**
  (исправлено в iOS 16.4+ при добавлении на домой, но не гарантировано)
- Работает только при добавлении "На экран «Домой»", не через вкладку Safari
- Лимит хранилища ~50MB (у нас используется ~3-5MB)

## Деплой и обновления

При каждом `npm run build` vite-plugin-pwa автоматически:
1. Генерирует список всех файлов с хэшами
2. Вшивает его в sw.js
3. При следующем открытии приложения у пользователей — SW обновляется автоматически

## Структура Service Workers

- `/sw.js` — основной SW, управляет кэшем и оффлайном (генерируется из public/sw.js)
- `/firebase-messaging-sw.js` — отдельный SW только для push-уведомлений
